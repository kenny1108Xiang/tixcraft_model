# Tixcraft Captcha Model

中文版本: [README_zh.md](./README_zh.md)

## Overview

A PyTorch-based captcha recognition model for 4-character lowercase English captchas from Tixcraft.

The pipeline covers synthetic captcha generation, dataset loading, model training on synthetic data, and finetuning on real captcha images. Recognition is treated as a fixed-length sequence prediction task — no character segmentation involved.

## Model Architecture

The model (`CaptchaTransformer` in `model.py`) works as follows:

1. Input image is converted to grayscale and resized to `32 x 128`
2. CNN backbone extracts visual features
3. Feature map is flattened into a sequence
4. Positional encoding is added
5. Transformer encoder models global context
6. Four learnable queries are passed into a Transformer decoder
7. Each query predicts one character position
8. Output shape: `(B, 4, 26)`

The model directly predicts all 4 positions in parallel, with 26 classes per position (lowercase a–z).

## Dataset Format

### Synthetic Data

Generated by `create_img.py`. Each split has image files and a `labels.txt`:

```text
abcd.png    abcd
wxyz.png    wxyz
```

### Real Data

Labels are parsed from filenames. All of the following map to label `abcd`:

```text
abcd.png
abcd_v2.png
abcd_001.png
```

Only labels with exactly 4 lowercase letters are accepted.

## Preprocessing and Augmentation

Handled in `dataset.py`. All images go through:

- grayscale conversion
- resize to `32 x 128`
- normalization (mean `0.5`, std `0.5`)

Training additionally applies:

- random affine transform
- brightness and contrast jitter
- Gaussian blur
- sharpness adjustment

## Training

### Step 1: Generate Synthetic Data

```bash
python create_img.py
```

Generates 50,000 images by default, split into `synthetic/train`, `synthetic/val`, and `synthetic/test`.

### Step 2: Pretrain

```bash
python train.py
```

Default settings:

- batch size: 256
- epochs: 50
- learning rate: `3e-4`
- optimizer: AdamW
- scheduler: OneCycleLR

Best checkpoint saved as `best_pretrain.pth`.

### Step 3: Finetune

Place real captcha images in `real_img/`, then run:

```bash
python finetune.py
```

Default settings:

- batch size: 64
- epochs: 60
- learning rate: `5e-5`
- optimizer: AdamW
- scheduler: CosineAnnealingLR
- early stopping patience: 15

Loads `best_pretrain.pth` and saves the best result as `best_finetune.pth`.

## Loss and Metrics

Cross-entropy loss is computed separately for each of the 4 positions. Total loss is the sum of all four.

Two metrics are tracked during training:

- **Full-sequence accuracy**: correct only when all 4 characters match
- **Character accuracy**: per-character correctness across all positions

## Limitations

- Fixed captcha length of 4
- Lowercase English letters only
- Fixed input size `32 x 128`

Not included in this repo:

- inference script
- ONNX / TensorRT export
- hyperparameters are hardcoded, no config file
